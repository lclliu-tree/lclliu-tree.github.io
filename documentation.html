What started as a laptop recommendation turned into a full day of productive system design. Here's what we built.

Hardware Setup

After a lengthy conversation about needs — PhD work in agent-based modeling, some gaming, portability, and a tight budget — we landed on a two-device setup:

  • ASUS TUF Gaming A15 (2020) — repurposed as a permanent desktop node after a bike accident bent the screen. Ryzen 7 4800H + GTX 1650, runs Linux VMs and heavy simulations.
  • ThinkPad E14 (incoming) — lightweight portable machine for office and campus use, connects to the TUF remotely.
  • iPad — on-the-go companion for reading, notes, and remote access.

Remote Access with Tailscale + xrdp

We configured the TUF as an always-on desktop node using Windows power settings. The plan is to run a Linux VM with xrdp and connect via Tailscale from both the ThinkPad and iPad. This means heavy compute is always reachable from anywhere without exposing ports publicly.

Git & GitHub Setup

Claude walked me through setting up Git and GitHub CLI on both Windows and WSL, generating SSH keys, and pushing three repositories:

  • Alignment_Game — replication code for my MAS paper, tagged v1.0-replication
  • NottenLiuTarasuk2025 — replication code for Notten, Liu & Tarasuk (2025), tagged v1.0-replication
  • alignment-game-abm — new Python package for network simulations of nonsymmetric alignment games (in development)

CV Auto-Compilation

We linked Overleaf (via CMU's institutional license) to GitHub, and set up a GitHub Actions workflow that automatically compiles my LaTeX CV to PDF on every push. No more manually exporting PDFs.
